# 第1课：认识 Git 和 GitHub

**预计时间：30分钟**

## 1.1 什么是 Git？

**Git** 是一个"版本控制系统"。简单说，它可以帮你：

- 📸 **拍照存档**：每当你完成一部分工作，可以用 Git 拍一张"快照"保存下来
- ⏪ **回到过去**：出了问题可以轻松回到之前的某个版本
- 🔀 **同时尝试多个方案**：创建"分支"在互不影响的情况下做不同的尝试
- 👥 **多人协作**：和团队同时开发一个项目，互不干扰

> **生活类比**：写论文时我们常保存"论文_v1.docx""论文_v2.docx""论文_最终版.docx"……Git 就是帮你自动管理这些版本的工具，而且功能强大得多。

## 1.2 什么是 GitHub？

**GitHub** 是一个存放 Git 仓库的"云盘"。它让 Git 可以：
- 把代码备份到云端
- 和他人分享代码
- 通过 Pull Request 协作

> **关系**：Git 是工具（装在你的电脑上），GitHub 是网站（用来存放和分享代码）。

## 1.3 安装 Git

### Windows 系统

1. 打开浏览器，访问 https://git-scm.com/downloads
2. 下载 Windows 版本
3. 双击安装，**所有选项保持默认**，一路点"Next"
4. 安装完成后，打开"命令提示符"或"PowerShell"，输入：

```bash
git --version
```

如果显示 `git version x.xx.x` 之类的信息，说明安装成功！

### 验证 Git 是否已安装

你的电脑可能已经装好了 Git。现在就来试试：

> **👉 动手操作：**
> 1. 打开终端（VSCode 里按 `` Ctrl+` ``，或者右键文件夹选择"在终端中打开"）
> 2. 输入以下命令：

```bash
git --version
```

## 1.4 首次配置

告诉 Git 你是谁——每次提交代码时都会记录这些信息：

```bash
git config --global user.name "你的名字"
git config --global user.email "你的邮箱@example.com"
```

> 用你的真实姓名或昵称都可以，邮箱建议用注册 GitHub 的邮箱。

验证配置是否成功：

```bash
git config --global --list
```

应该会看到你刚设置的名字和邮箱。

## 1.5 常用术语（先混个眼熟）

| 术语 | 意思 |
|------|------|
| **仓库/Repository** | Git 管理的项目文件夹 |
| **提交/Commit** | 一次"拍照存档" |
| **分支/Branch** | 独立的开发线路 |
| **工作区** | 你正在编辑的文件 |
| **暂存区/Stage** | 提交前的"预备区" |

> **现在不用记！** 后面每课都会用到，自然就记住了。
